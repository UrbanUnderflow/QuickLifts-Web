# Publish a daily Macra KPI snapshot from Scoreboard, Experiments, purchase logs, cancel reasons, retargeting, and AppsFlyer coverage

## Core narrative

The daily Macra KPI snapshot is derived from the existing `Scoreboard`, `Experiments`, purchase logs, cancel reasons, user state, retargeting state, and AppsFlyer imports before any partner-facing messaging is published. That means the operating picture is assembled from those live source artifacts first, and only then translated into any external summary, funnel interpretation, or partner-facing update. If that daily snapshot has not been built from those inputs, the messaging should wait.

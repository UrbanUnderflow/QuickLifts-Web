# Publish a daily Macra KPI snapshot from Scoreboard, Experiments, purchase logs, cancel reasons, retargeting, and AppsFlyer coverage

## Core narrative

The daily Macra KPI snapshot must be assembled from the existing Scoreboard, Experiments, purchase logs, cancel reasons, retargeting state, and AppsFlyer coverage before any partner-facing interpretation is shared. That means the operating view is built from the real source inputs first, then any external summary, partner update, or funnel narrative is written from that assembled snapshot rather than from stale takeaways, isolated metrics, or unsupported assumptions.

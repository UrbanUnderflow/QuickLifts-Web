# Refresh active variant_a experiment results before making funnel partner messaging

## Core narrative

Active `variant_a` experiment results must be refreshed from the current operating inputs before any funnel-partner messaging decision is made. That means partner-facing funnel copy, campaign framing, and trial-start messaging do not move forward from stale assumptions, old summaries, or prior recommendations that have not been checked against the live operating view. We refresh the active `variant_a` read first, then decide what the message is allowed to claim.

## Messaging pillars

### 1. Scoreboard and Experiments define the safe story

The first constraint on partner-facing messaging is the live read across **Scoreboard** and **Experiments**. If refreshed active `variant_a` performance is still holding, the message can reinforce the current funnel story. If it is softening, we do not compensate with louder language. We keep the story inside what the current Scoreboard and Experiments state can actually support.

### 2. Purchase logs, cancel reasons, user state, and retargeting state explain the friction

**Purchase logs** show whether the funnel is converting, while **cancel reasons**, **user state**, and **retargeting state** help explain where trust is breaking down or follow-up is missing the moment. Those inputs tell us whether the message is clear, whether the ask is arriving too early, or whether the handoff after interest is mismatched to the user’s real position. Messaging should answer the friction those systems are surfacing, not a guessed version of it.

### 3. AppsFlyer imports keep scale claims honest

**AppsFlyer imports** help separate a real messaging win from a source-quality illusion. If refreshed active `variant_a` results are strong only for a narrow slice of traffic, we should not stretch that into a broad partner claim. Reading AppsFlyer imports alongside the refreshed experiment state keeps the funnel story precise and shows which message can scale safely versus which one only works under a thinner acquisition mix.

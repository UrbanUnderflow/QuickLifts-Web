# Setting Up Firebase Collections

To support the Virtual Office, the following collections must be created or configured within your Firestore instance. The Web UI and Agent Runners depend heavily on these schemas.

## 1. `agent-presence`
This collection tracks the "online" status of an agent. An agent updates its own document periodically (heartbeat).
*   **Document ID**: The agent's unique name (e.g. `nora`, `scout`, `antigravity`)
*   **Fields**:
    *   `displayName` (String): Human readable name.
    *   `status` (String): e.g., "Online", "Working", "Offline".
    *   `emoji` (String): A visually distinct icon used in the UI.
    *   `currentTask` (String): What the agent is currently doing.
    *   `lastUpdate` (Timestamp): The time of the last successful heartbeat. 

## 2. `agent-kanban`
This acts as a shared to-do list for human operators to dispatch tasks to specific agents.
*   **Document ID**: Auto-generated by Firestore.
*   **Fields**:
    *   `title` (String): The objective or goal.
    *   `description` (String): The detailed instructions or prompt.
    *   `status` (String): "backlog", "in-progress", "done"
    *   `assignee` (String): The agent document ID this task belongs to.
    *   `createdAt` (Timestamp)

## 3. `agent-logs`
Instead of looking at a terminal, agents write their `stdout` and `stderr` streams directly to this collection. The Virtual Office UI reads this in real-time.
*   **Document ID**: Auto-generated by Firestore.
*   **Fields**:
    *   `agentId` (String): The agent the log belongs to.
    *   `message` (String): The log content.
    *   `level` (String): "info", "warn", "error".
    *   `timestamp` (Timestamp).
